# Built to Outlive the Algorithm: Hybrid Post-Quantum Cryptography and Crypto-Agility in a Sovereign Messaging Stack

## One-Line Summary
We migrated a sovereign messaging stack (skchat/skcomms/capauth) to hybrid post-quantum crypto — surface by surface, secure-if-either-leg-holds — but the real contribution is the *scaffolding*: self-describing crypto suites + pluggable backends + a self-report that refuses to overclaim, so the next quantum algorithm plugs in without rewriting the apps.

## The Core Finding
- **The threat is harvest-now-decrypt-later, not Q-Day.** Mosca's Inequality: for 10–20-year secrets, `X(secrecy) + Y(migrate) > Z(years-to-CRQC)` *today*. So **confidentiality** (retroactively breakable) is migrated first; **signatures/identity** (forgery only, not retroactive) are a deliberate Phase 2; **symmetric** (AES-256/SHA-2) is already fine and we refuse to fear-monger it.
- **One hybrid construction, everywhere.** `HKDF-SHA256(X25519_ss ‖ ML-KEM-768_ss)` — concatenate-then-KDF, never XOR, never pure-PQ — secure if *either* leg holds. The same `x25519-mlkem768` (FIPS 203) used by TLS and Signal PQXDH. We never hand-roll the math (liboqs / noble / pyca); a missing PQ backend is a *loud error*, never a silent downgrade.
- **Surface by surface:** hybrid KEM; `pqdm1:` DM/envelope sealing with a downgrade-lock AAD; a **Level-3 1:1 DM epoch-ratchet** (`dm_ratchet.py`/`dm_manager.py`, FS + PCS) **proven live cross-node between two physical machines** (`lumina@.158` ↔ `jarvis@.41`, over tailscale federation — each `DmRatchetManager` sealed a DM the other opened, hybrid throughout); a **per-epoch group ratchet** (FS + PCS) that kills the static `os.urandom(32)` group key — the highest-leverage HNDL bug; hybrid at-rest key-wrap; an Ed25519+ML-DSA-65 **AND-gate** signature; and a Sequoia-issued **post-quantum signing root** (v6 ML-DSA-87+Ed448) for the Lumina *agent* root — **generated + cross-signed old↔new (WoT-verified)**. *Cryptographic ceremony complete; the live signing cutover is in progress (gated on PGPy→`sk_pgp`); operator root + vault re-seal sequenced after — the live identity is NOT yet migrated.*
- **Two sovereign libraries:** **`sk_pqc`** — the hybrid KEM, **now published to the public registries across Python, Dart, and Rust behind one wire format** ([PyPI `sk-pqc`](https://pypi.org/project/sk-pqc/) / [pub.dev `sk_pqc`](https://pub.dev/packages/sk_pqc) / [crates.io `sk-pqc`](https://crates.io/crates/sk-pqc), crates.io cut 2026-06-27; all import `sk_pqc`; repos `sk-pqc-{py,dart,rs}`); the Dart package delivers hybrid PQC **in the browser** (where no WebCrypto PQC API exists), CDP-verified in-page DMs. **In progress: a single PyO3 Rust-core** (`sk-pqc-rs --features python` → abi3 `sk_pqc_rs` wheel, pure-Rust X25519+ML-KEM-768, no OpenSSL/liboqs) so **one vetted implementation backs every language binding** — selected by id, wire format frozen. Experimental & unaudited. **`sk_pgp`** (Python, PyO3→Sequoia) — the **PGPy replacement** and the *engine* that lets the Python services sign with v6/PQC keys at all (PGPy can't parse v6, gpg can't sign PQC); in-process ML-DSA-87+Ed448 generate/sign/verify proven, self-contained wheel.
- **Headline: a runtime identity-mode switch (DID ↔ anonymous).** The *same* hybrid KEM + signature-free ratchet run under either a **sovereign / attributable** mode (capauth **DID** + hybrid Ed25519+ML-DSA-65 signed prekeys — federated, auditable) or an **anonymous / deniable** mode (no DID, opaque queue ids, repudiable HMAC — SimpleX-style), chosen **per-conversation** with a **per-deployment default**. Invariants: *sovereign never silently downgrades*, and *ratchet steps stay signature-free in BOTH modes* so content deniability survives even in sovereign mode (identity is asserted only at session establishment). So PQC confidentiality + metadata privacy are decoupled from *whether you are identified at all*. **Honest status:** scaffolded (anon-queue + signed-prekey primitives exist), mode-flag plumbing + invariant enforcement in progress (coord `c0f105a1`), default-OFF, not yet on the live transport.
- **The thesis — crypto-agility, vindicated in the field:** a `suite_id` on every object, a `CryptoBackend` ABC (PGPy/GnuPG/Sequoia-CLI/`sk_pgp`), KEM selected `by suite id`, and a 4-step recipe (register suite → add backend → bump wire version → negotiate) to adopt the next scheme **with zero app changes**. Not just argued — *watched hold*: the same `x25519-mlkem768` suite, selected by id, carried a Level-3 DM ratchet between two physical machines over tailscale with zero app changes, and three published `sk_pqc` packages negotiate it byte-for-byte across three languages. Our infra rides on top of whatever comes next.
- **Honesty as a feature:** a self-report engine (`pqc_report.py`) that *structurally cannot* mark a surface quantum-resistant unless its live suite is — unknown suites resolve classical, a downgrade shows up classical, and the root is reported classical until the day it isn't.

## Method / Evidence
- Real constructions, real sizes, real vectors: a cross-impl KEM vector anchored to **NIST ACVP FIPS 203**; bidirectional Python↔Dart DM interop; **a Level-3 forward-secret hybrid-PQ DM ratchet proven live cross-node** between `lumina@.158` and `jarvis@.41` over tailscale federation (each box's `DmRatchetManager` sealed a DM the other opened); the **agent-root ceremony run on a real key** (v6 `mldsa87-ed448` generated + cross-signed old↔new, continuity verified); `sk_pgp` in-process ML-DSA-87+Ed448 gen/sign/verify; three `sk_pqc` packages live on PyPI/pub.dev/crates.io; ~68 PQ pytest functions + Dart KATs.
- Honest scope: confidentiality is **hybrid where negotiated**; the **cross-node Level-3 ratchet was proven via the `dm_manager` directly** — the end-to-end round-trip through the live production daemon is a **pending belt-and-suspenders** check; the one-core PyO3 binding is **mid-build**; signatures/identity are **hybrid-available, opt-in**; the agent-root **cryptographic ceremony is complete but the live identity is NOT yet migrated** (cutover in progress, gated on PGPy→`sk_pgp`; operator root + vault re-seal after); the comms tier is deliberately **-768, not CNSA-2.0**. No "quantum-proof" claim is made anywhere.

## Conclusion
The durable win is not ML-KEM-768 or ML-DSA-65 — it is an infrastructure where the choice of post-quantum scheme is a registry entry and a pluggable backend, every object self-describes what protects it, and overclaiming is structurally impossible. Built to outlive the algorithm.
